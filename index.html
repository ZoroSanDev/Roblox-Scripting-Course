Understanding Basic Data Types

Data types represent different forms of information. Consider the following example:

What color is an apple?

  1. Blue
  2. Purple
  3. Red
  4. Black

For this question, a text-based response is required, such as "Red".

On the other hand, some questions may demand a numerical response:

4 + 5 = ?

  1. 8
  2. 1
  3. 6
  4. 9

Here, a numerical answer is expected, not text.

These examples highlight the diverse use of data types in our daily activities.

Primitive Data Types in Roblox

Roblox recognizes four primitive data types:

  1. nil - no value
  2. boolean - true / false
  3. number - a numeric value
  4. string - text

In programming, variables store these data types. Think of variables as you would in mathematics, where "x" represents a number:

local x = 16 - 7
local x = 9

Here, the letter "x" stores the number 9.

When storing a data type in a variable in Roblox, you must declare it. The process is simple:

  1. Declare the variable as a local variable
  2. Assign a name to the variable (similar to "x," but choose a name that reflects the stored value)
  3. Assign a value to the variable

Now that we understand variable declaration, let's explore basic operations.

Basic Operations with Data Types

It's essential to recognize that different data types have different methods.

With a number data type, we can perform calculations:

local x = 5
local n = 10
local sum = x + n
local mult = x * n
local div = n / x
local pow = x^2

print(sum) -- 15
print(mult) -- 50
print(div) -- 2
print(pow) -- 25

With a string variable, we can concatenate strings:

local text1 = "Hello, "
local text2 = "how are you?"
local multiLineText = [[Multi
Line
String]]

local sentence = text1 .. text2

print(sentence) -- Hello, how are you?
print(multiLineText) -- Multi
-- -------------------- Line
-- -------------------- String

Boolean and nil data types have fewer options:

local boolean = true
local nilValue = nil

print(boolean) -- true
print(nilValue) -- nil

boolean = not boolean -- true becomes false
print(boolean) -- false
boolean = not boolean -- false becomes true
print(boolean) -- true

Additional Insights (To Be Covered Later, After Teaching Functions)

  1. Understanding Variables

Variables can have global or local scopes. They have global scope by default, but it's almost always better to create them with local scope because Luau accesses local variables faster than global ones. To give a variable local scope, put the keyword local before a variable's name when you assign a value to it.

A local variable is accessible only in the block where itâ€™s declared. A (non-local variable) is visible to all scopes of a script.. Consider the following example:

function printX()
    print(x)
end

local x = 5
printX()

This code will print nil. (Because x is not defined before the function)

Conversely, the following code will work:

function printX()
    print(x)
end

x = 5
printX()

When a variable is not defined as local, it becomes global. Even if a function is written before the declaration, as long as it runs after setting a value, the function will obtain the value of the variable.
